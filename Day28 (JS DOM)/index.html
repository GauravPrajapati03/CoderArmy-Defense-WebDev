Selecting the elements of html

  • .getElementById
  • .getElementByClassName
  • .getElementByTagName
    Using CSS Selectors
  • .querySelector (Selects the first one)
  • .querySelectorAll

When we use element selectors we get the array of HTMLCollections but when we use css selectors then we get array of NodeList

        Reason:

    1 Element Selectors Examples: 
    document.getElementsByTagName('div') 
    document.getElementsByClassName('box')
    document.getElementsByName('username')
        
        *These methods return a live HTMLCollection.
        *A live collection automatically updates when the DOM changes (e.g., if you add or remove matching elements).

    2 CSS Selectors Examples:
    document.querySelectorAll('.box')
    document.querySelector('#main')
        
        *These methods use CSS-style selectors.
        *querySelector() returns a single element (the first match).
        *querySelectorAll() returns a static NodeList.
        *A static collection means it does not update automatically when the DOM changes.


    So Why the Difference?
    
        1) Itâ€™s mostly historical and design-based:The older DOM Level 1/2 APIs (getElementsByTagName, etc.) were made before CSS selector support existed. They were implemented to give a "live view" of the DOM.
        2) The newer DOM Level 3 Selectors API (querySelector, querySelectorAll) was introduced later, focusing on CSS-style querying.
        3) For performance and predictability reasons, it returns static results instead of live ones.

*** What "live" means: If you add a new <p> to the page after you've selected them, the allParagraphs collection will automatically update to include it.***
