File Path

File paths in HTML specify the location of files (like images, stylesheets, scripts) that a webpage needs to load. Paths can be either absolute or relative, and understanding both is crucial for correct linking in web development.

Types of File Paths

  1. Absolute File Path

    • Specify the full URL or full path to the resource.
    • Independent of the location of your HTML file.
    • Useful when the resource is hosted externally.
    <img src="https://example.com/image.jpg">

  2. Relative File Path

    • Specify the path relative to the current HTML file.
    • Useful when the resource is hosted locally.
    • Can navigate through directories using special notations.
    <img src="../images/photo.jpg">

Directory Navigation Symbols

  • ./ refers to the Current Directory

  • ../ refers to the Parent Directory

  • / Root Directory

Why does Browsers do not allow showing images using absolute file?

Browsers do not allow showing images using absolute file paths referring to local files 

(like file:///C:/images/pic.jpg)
due to security restrictions built into modern web browsers. Hence, Absolute local file path references are blocked in browsers to protect user security and privacy, enforce proper web sandboxing, and ensure portable, consistent web content delivery.

HTML Boilerplate Code

An HTML boilerplate is a basic template or starting structure for an HTML document that includes essential elements and best practices to ensure proper rendering and compatibility in browsers.

Purpose of HTML Boilerplate

  • Provides a standardized foundation for new web projects.
  • Includes necessary HTML5 elements, meta tags, and structure.
  • Saves time and ensures consistency across multiple pages or projects.
  • Helps maintain best practices for accessibility, responsiveness, and SEO.
  • Ensures your pages render correctly on all browsers and devices.
  • Sets up responsiveness for mobile and varied screen sizes.
  • Follows modern HTML5 standards and accessibility guidelines.
  • Provides a solid, maintainable starting template for faster development.

Explanation

  • <!DOCTYPE html>: Declares the document type and HTML version (HTML5 here).
  • <html lang="en">: Root element with language attribute for accessibility and SEO.
  • <head>: Contains metadata like character set, viewport for mobile, title, and links to CSS and favicon.
  • <body>: Contains the visible content of the webpage.
  • Linking CSS and JavaScript files enables separating styling and behavior from content.

Semantic Elements

Semantic elements clearly describe their meaning for both browsers and developers, improving accessibility, SEO, and code clarity.

<header> Tag

  • Represents introductory content or a group of navigational links.
  • Typically contains site logo, title, main navigation menu.
  • Can be used multiple times on a page (for sections/articles).

<footer> Tag

  • Defines a footer section for a document or section.
  • Usually contains contact info, copyright info, links, author details.
  • Can also be used multiple times on a page.

Other Common Semantic Elements

Tag Purpose
<main> Represents the main content of the page (only one per page)
<nav> Contains major navigation links
<section> Defines sections of content grouped thematically
<article> Represents independent content like blog posts or news articles
<aside> Content tangentially related to main content (e.g., sidebars, ads)

Div, span, id, class

  1. <div> Tag
    • A generic container for grouping other HTML elements.
    • It has no meaning by itself and is used for styling or scripting.
    • Typically used to create sections or wrappers in layouts.
  2. <span> Tag
    • Inline container similar to div, but it does not start a new line.
    • Used to style or target small parts of text or inline elements.
  3. id and class Attributes
    • id uniquely identifies a single element on the page. Should be unique per page.
    • class groups multiple elements that share similar styles or behaviors.
    • Both are used in CSS and JavaScript for targeting elements.
